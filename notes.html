The 5 Units

Unit 1 — Agents & Vibe Coding SDLC

  • Big idea: generation is solved — the new craft is verification, judgment, direction.
  • Agent = Model + Harness. Most agent failures are config failures, not model failures.
  • Vibe coding → agentic engineering. Differentiator = verification: tests (deterministic) + evals (LM-judged).
  • You're the orchestrator, not the typist. SDLC: Plan, Design, Implement, Test, Deploy, Maintain.
➡ Capstone: you direct + verify the agent; that's the whole pattern.

Unit 2 — Tools & Interoperability ("10x Agents")

  • Big idea: open protocols kill the tech debt of custom tool integrations.
  • MCP (Model Context Protocol) — connects models to tools/data. A2A — agent-to-agent. A2UI — generative UI. AP2/UCP — machine-to-machine payments/commerce.
➡ Capstone: your scripts wrapped as an MCP server = the agent's tools.

Unit 3 — Agent Skills (Memory & Context)

  • Big idea: avoid "context rot" with portable Agent Skills — folders built around a central SKILL.md.
  • Progressive disclosure: keep the prompt light; load tools/details only on demand → one agent flexes into hundreds of specialist roles.
➡ Capstone: give each agent (score/match/personalize) its own skill.

Unit 4 — Security & Evaluation key day

  • Big idea: build "Effective Trust" via a 7-pillar security architecture for non-deterministic AI.
  • Tactics: ephemeral sandboxing, defense vs "slopsquatting" (fake hallucinated packages), a Red/Blue/Green security triad, OpenTelemetry trajectory evals, and human-in-the-loop triage.
➡ Capstone: your "nothing sends without approval" gate = textbook human-in-the-loop security.

Unit 5 — Spec-Driven Production (SDD)

  • Big idea: bridge fragile prototypes to production. Code is disposable; behavior specs (Gherkin) are the source of truth.
  • Zero-trust pipelines, automated code-review agents, Policy Servers, deploy to cloud (Cloud Run) + observability.
➡ Capstone: a clean README + reproducible setup covers the "production" expectation.

🔌 What is MCP?

MCP = a USB-C port for AI. Instead of hand-wiring a custom connector every time an agent needs a tool, you wrap the tool once as an "MCP server" and any agent can plug in. It's how your agent reaches your real-estate scripts.
